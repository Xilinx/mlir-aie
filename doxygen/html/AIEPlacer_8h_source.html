68// Sequential placement algorithm
69//
70// Greedy, single-pass placer that maps each `aie.logical_tile` (LTO) to a
71// physical (col, row). Four phases:
72//
73// 1. Collect LogicalTileOp / ObjectFifo* / Cascade* / Flow* / PacketFlow*
74// from the device.
75// 2. Build per-LTO constraints: buffer/cascade/compute-peer adjacencies,
76// channel requirements, and needNeighborIn/Out (the minimum number of
77// compute peers per LTO that MUST land on a shared-L1 neighbor to fit
78// the per-tile DMA budget).
79// 3. Place compute tiles. Sort by constraint level (pinned -> partial ->
80// unpinned) then by descending placementPriority. For each LTO:
81// validate buffer/cascade/compute-peer constraints and channel budget,
82// then claim the tile. Unpinned candidates are tried in two passes:
83// first the slots not soft-reserved for another demand-bearing LTO's
84// peers, then a fallback pass that accepts reserved slots. Each
85// placement of a demand-bearing LTO soft-reserves its mem-affinity
86// neighbor slots for that LTO's compute peers.
87// 4. Place each remaining non-core (mem/shim) LTO near the column
88// centroid of its placed-core peers. With mergeLogicalTiles == true
89// (default) several non-core LTOs may share one physical tile when
90// DMA capacity allows; with mergeLogicalTiles == false each non-core
91// LTO claims its own physical tile.
92//
93// Greedy: once a tile is chosen the decision is final, with no backtracking.
94// If a later LTO's constraints become unsatisfiable given prior placements,
95// the placer emits an error pointing at the LTOs the user can manually pin
96// to break the deadlock.
97//
98// Compute-to-compute ObjectFifos consume no DMA channel when their endpoints
99// land on shared-L1 neighbor tiles, so they enter the placer as a separate
100// compute-peer adjacency rather than as channel demand.
101class SequentialPlacer : public Placer {
102public:
103 SequentialPlacer(std::optional<int> coresPerCol = std::nullopt,
104 bool mergeLogicalTiles = true)
105 : coresPerCol(coresPerCol), mergeLogicalTiles(mergeLogicalTiles) {}
106
107 void initialize(const AIETargetModel &targetModel) override;
108
109 mlir::LogicalResult place(DeviceOp device) override;
110
111 llvm::StringRef getName() const override { return "sequential_placer"; }

185 int neighborDemand(mlir::Operation *op) const {

329 // Compute-peer DMA budget check. Returns true if placing `logicalTile` at
330 // `candidate` keeps the compute-peer neighbor demand satisfiable for BOTH
331 // the LTO being placed AND every already-placed compute peer of the LTO.
332 // A compute LTO with peer demand N (N compute-peer fifos must use shared
333 // neighbor memory to fit DMA budget) has slack (totalComputePeers − N): it
334 // can afford at most that many non-neighbor compute peers. The check also
335 // forward-looks: unplaced peers still need a free physical compute-tile
336 // neighbor of `candidate` to land on.
